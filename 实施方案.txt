JoyPilot 实施方案 MASTER PLAN v2.2（完整版）

AI-first / Gate-driven / Radar Console / ShopAppealLab Gate Pack / Aquarius Protocol

0. 最高原则（写死，不讨论）
0.1 Contract-first

先定义输入/输出契约（Pydantic/JSON Schema）→ 再写 fixtures（金样/红队/负向）→ 再实现。

0.2 Deterministic-by-default

输出结构、字段顺序、枚举、排序规则 全部确定化

允许“多样性”的地方必须 seed 固定（同输入同输出）

LLM 只允许在指定字段内润色，不得改结构/意图/阈值结论

0.3 No silent failure

宁可返回“高熵/不确定，需要澄清/需要用户确认”，也不允许编造确定结论。

0.4 Safety & Respect（去 PUA 化工程约束）

禁止输出：胁迫/羞辱/操控/纠缠；惩罚式冷落、内疚绑架、逼问逼表态、阴阳怪气贬损；任何“制造焦虑”“测试服从/边界”的意图或话术。

0.5 Consent-first for Sensitive Context（敏感语境同意优先）

经期/健康等敏感语境：

仅 Premium 可用（权限门禁）

用户每次会话显式确认后才启用（同意门禁）

不做健康推断

默认不点名（除非用户允许且来源明确）

不入记忆/不入日志原文（脱敏门禁）

0.6 Self-stability First（自我稳定优先）

当压力/不确定性/施压风险升高时，系统优先输出：降速、澄清、停止施压、转移注意力到自我生活（产品机制，而非口号）。

0.7 Aquarius Protocol（“水瓶座补丁包”总原则）

均值回归平滑：短期回落不等于崩盘

现实结界覆盖：命中家人/会议/睡觉/重要客户等 → 强制 OFFLINE_SOCIAL_SHIELD

框架自洽锁：用户已锁定“低压力表达” → 禁止生成索取/抱怨不回的矛盾文本

沉没成本熔断：物质投入触发 48h 暂停加码（自我预算控制，不做交易式逼回馈）

1. 护城河定义（产品层）

JoyPilot 护城河 = User Voice DNA（像用户本人） + Per-Contact Dossier（关系档案） + 状态机（熵/回滚） + 雷达监控台（可解释动作） + 自动门禁留证（可控可审计）。

7. Aquarius Protocol（四项增强：写死为门禁可测规则）
A1（J3）：Mean Reversion Smoother（均值回归）

Spike 检测（过去 24h 极端峰值）→ 24–48h 内 mean_reversion.active=true

UI 强制输出：“均值回归期，勿追问/勿连发”（固定模板）

若出现明确拒绝/边界 → 不走均值回归安抚，直接边界策略

A2（J11）：Physical Shield Detection（现实结界覆写）

Briefing/对话命中现实高优先事项类别词 → 无视高甜积累，强制 OFFLINE_SOCIAL_SHIELD + 24h cooldown + Focus Redirect（固定模板）

A3（J10/J7）：Frame Integrity Lock（框架防崩塌锁）

dashboard 中 frame_anchor：OFF|CANDIDATE|LOCKED

自动识别只能到 CANDIDATE；用户确认后才 LOCKED

LOCKED 后如果用户生成抱怨不回/索取回应 → frame_violation 拦截 + 给低压力替代

A4（J12）：Sunk Cost Circuit Breaker（沉没成本熔断）

标记/明确证据出现外卖/礼物 → breaker active 48h

UI 常驻红提示：暂停物质投入与长文输出，48h 后再评估（固定模板）

16.1 run_and_log 的真实行为与参数（写进 USAGE）

USAGE

16.2 重要限制：run_and_log 会固定传 --base_url

维护指南明确：run_and_log 运行 smoke 时固定执行 python <smoke_script> --base_url http://127.0.0.1:PORT，所以所有要被 run_and_log 包裹的 smoke 必须能 parse --base_url（即使离线不用）。对离线 smoke 的现实处理只有两种：要么手工按 log 模板追加（必须脱敏），要么给 smoke 补 --base_url 兼容再统一留证。JoyPilot 直接继承这条硬规则。
